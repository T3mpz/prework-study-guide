Hyper Text Markup Langauge (HTML)

  • HTML provides structure for a webpage.
  • Elements are used to divide content on the page.
  • Semantic HTML elements describe content and the role it plays on a webpage.
    • Ex. Head, Body, Header, Main, Section, and Footer.
  • Head element contains information (metadata) which helps the browers render correctly.
    • Metadata is data about our webpage that the browser uses to effectively serve our webpage to users
  • Attributes define additonal information about an element.

Cascading Styling Sheet (CSS)

  • CSS is a programming language used by developers to define how webpage content is presented to users.
  • A margin property indicates how much space we want around the outside of an element.
  • The padding property adds space around the content inside an element.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • A box shadow is described by X and Y offsets relative to the element, blur and spread radius, and color.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.

Git & GitHub

  • Git is a version control system that allows you to merge and track different versions of code.
  • GitHub is a cloud-based hosting service that allows us to track web projects inside remote repos.
  • Version control - Version control systems allow us to track and manage changes to our codebase.
  • Branch - workspace where you can make changes; avoiding changes to live site.
  • Commit (changes) - add/edit/delete a new version of a file you're working on.
  • Pull request (PR) - Changes on your branch will merge into another branch (often main(MASTER) branch).
  • Merge - when a pull request (PR) is approved, changes are merged into the main branch.
  • GitHub issues - gives developers and teams an easy way to keep track of key tasks and progress.
  • Feature branch - When we create a new branch to add something new to our project, we call it a feature branch.
  • Base branch - where changes (commits) will be merged into; (default branch/main).
  • Git clone - Clones a newly created directory.
  • Git pull - Incorporates changes from a remote repo into current branch.
  • Git checkout -b - Creates a new brnach.
  • Git add -A - Adds all modifications to staging.
  • Git commit -m - Creates a new commit with a message about changes.
  • Git push - Pushes any changes locally into remote repo.

JavaScript

  • JavaScript (JS) is a high-level dynamic programming language that allows users to interact with websites.
  • Control flow is the order in which a computer executes code in a script.
  • A variable is a named container that allows us to store data in our code.
  • Start by declaring a variable with the let keyword, followed by the name you give to the variable.
  • A semicolon at the end of a line indicates where a statement ends.
  • An operator is a mathematical symbol that produces a result based on two values (or variables)
  • Conditionals are code structures used to test if an expression returns true or not.
  • We can use conditional statements and loops to control the order in which code is executed, or if it is even executed at all.
  • Functions are a way of packaging functionality that you wish to reuse
  • Method - a set of instructions that can be executed by a computer
  • An array is a single variable that is used to hold a group of data
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
